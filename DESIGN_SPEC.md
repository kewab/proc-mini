# 采购小程序设计规范（v1.0）

> 生效范围：`/src` 全量页面与组件。
> 生效日期：2026-04-19。
> 目标：统一视觉语言、交互反馈与布局规则，后续开发按本规范执行。

## 1. 设计原则

1. 信息优先：采购关键数据（价格、数量、状态）优先级最高。
2. 操作明确：所有可点击区域必须有按压反馈（遮罩或 Ripple）。
3. 一致可复用：同类组件（按钮/卡片/角标/导航）样式与状态保持一致。
4. 轻量高对比：浅底+高可读文字，重点行为使用品牌蓝，价格/告警使用橙红。
5. 面向小程序：优先保证触达范围、safe-area、弱网与低性能机型稳定性。

## 2. Design Tokens

## 2.1 色彩

- 品牌主色（Primary）：`#2EA8FF`
- 主色高亮（Tab 选中）：`#0055CC`
- 主色浅底（选中背景/弱强调）：`rgb(0 85 204 / 10%)`
- 危险/价格主色：`#FF5A36`
- 危险深色：`#E64524`
- 页面背景（默认）：`#F6F7FB`
- 页面背景（次级）：`#F5F5F5`
- 卡片背景：`#FFFFFF`
- 一级文字：`#111827` / `#333333`
- 二级文字：`#6B7280` / `#8E8E93`
- 辅助文字：`#9CA3AF`
- 分割线：`#EDF0F4` / `#ECECEC`
- 禁用背景：`#BFDBFE`

规则：

- 新增颜色必须先映射到语义角色（如 `color-brand-primary`、`color-text-secondary`），禁止无语义随意加十六进制值。
- 价格、删除、告警统一使用危险色系，不与品牌蓝混用表达语义。

## 2.2 字体与字号

- 字体栈：`'HarmonyOS Sans', 'SF Pro Text', 'PingFang SC', 'Microsoft YaHei', sans-serif`
- 推荐字号（rpx）：
- `20`: 辅助说明/角标
- `22-24`: 次要正文/标签
- `26-30`: 常规正文/按钮文案
- `34`: 页面标题
- `40-44`: 价格与关键数字

规则：

- 单个组件内字号阶梯不超过 3 级。
- 价格信息必须至少比同区块普通文本高 8rpx。

## 2.3 间距与圆角

- 基础间距单位：`4rpx`
- 页面左右内边距：`16rpx / 18rpx / 20rpx / 28rpx`（按页面密度选择其一）
- 组件常用间距：`8rpx / 12rpx / 16rpx / 24rpx`
- 圆角：
- 小圆角 `8rpx`
- 中圆角 `16rpx`
- 大圆角 `20rpx / 24rpx`
- 胶囊圆角 `999rpx`

## 2.4 阴影层级

- 卡片阴影：`0 12rpx 28rpx rgb(15 23 42 / 8%)`
- 底部浮层/栏：`0 -8rpx 30rpx rgb(31 45 61 / 6% ~ 12%)`
- 弹层阴影：`0 0 16rpx rgb(0 0 0 / 8%)`

## 2.5 动效

- 点击反馈：`180ms ~ 220ms`
- 页面内过渡：`200ms ~ 300ms`
- 功能型动效（加购飞球）：`560ms ~ 620ms`
- Tab 选中上浮：`8rpx`

规则：

- 交互动效必须“短、可感知、不打断任务”。
- 禁止引入无业务意义的长动画。

## 3. 布局规范

## 3.1 安全区

- 所有底部固定区（Tabbar、结算栏、购物车汇总栏）必须包含：
- `padding-bottom: env(safe-area-inset-bottom)`
- `padding-bottom: constant(safe-area-inset-bottom)`

## 3.2 顶部导航

- 自定义导航高度按设备计算：状态栏 + 胶囊按钮 + 上下补偿。
- 页面标题居中，左右按钮命中区域不小于 `64rpx x 64rpx`。

## 3.3 底部区域

- Tabbar 固定高度：`106rpx + safe-area`
- 存在购物车汇总条时，页面底部预留需额外增加 `112rpx` 以上。
- 固定底栏与内容区必须避免遮挡，使用 layout 统一兜底。

## 4. 组件规范

## 4.1 Tabbar（自定义）

- 图标 + 文案上下结构，所有 Tab 均显示文案。
- 未选中：灰色文案（`#8E8E93`）。
- 选中：蓝色文案（`#0055CC`），图标上浮 `8rpx`。
- 点击反馈：
- 必须有半透明遮罩；
- 推荐 Ripple（当前已实现 `tabbar-ripple`）。

## 4.2 按钮

- 主按钮：品牌蓝底白字，圆角胶囊。
- 次按钮：浅色底+品牌色字或纯文本按钮。
- 禁用态：降低对比度并禁用关键行为触发。
- 最小点击高度建议 `72rpx`。

## 4.3 商品卡片

- 结构：图片、标题、副文案、价格区、操作区。
- 标题最多两行；价格必须与单位分层展示。
- 购买入口（购物车按钮）需固定尺寸、固定位置，避免布局抖动。

## 4.4 角标/数量徽标

- 角标内容超过两位时显示 `99+`。
- 必须设置最小宽度和上限宽度，文本居中。
- 角标相对父容器右上角偏移，禁止挤压主内容。

## 4.5 表单与步进器

- 输入框边框与按钮保持统一色值（`#ECECEC` 系）。
- 减号/加号按钮触达区域必须一致。
- 数量输入仅允许正整数，空值回退 `1`。

## 4.6 弹层与提示

- 使用 `t-dialog`、`t-toast` 作为统一反馈通道。
- 破坏性操作（如删除）必须二次确认。
- Toast 文案简短明确，避免长句。

## 5. 页面级规范

## 5.1 首页

- 顶部氛围背景可使用模糊图/渐变，但正文区域必须保证阅读对比度。
- 网络异常卡片使用统一错误色系与重试按钮。

## 5.2 分类页

- 左右结构：左侧类目、右侧商品流。
- 顶部筛选/标签区域固定视觉层级，避免与列表内容混色。
- 加购按钮与数量角标遵循组件规范 4.3、4.4。

## 5.3 购物车页

- 结算栏固定底部，金额和下单按钮保持最高视觉权重。
- 空态需包含说明与清晰跳转动作（去逛逛）。

## 5.4 我的页

- 头部可有装饰背景，但功能入口区（订单/工具）须保持卡片化与高可读。
- 用户信息、权益卡、推荐区间距保持节奏一致。

## 6. 实现约束（开发必须遵守）

1. 优先复用现有组件，不重复造视觉近似组件。
2. 样式优先使用 Tailwind 原子类 + 少量语义 class；复杂样式放 scoped scss。
3. 新组件必须包含 `默认 / 按压 / 禁用` 状态定义。
4. 颜色、字号、圆角、阴影必须来自本规范，不得随意漂移。
5. 所有固定定位组件必须经过 safe-area 校验。
6. 涉及数量展示必须处理上限文案（如 `99+`）。
7. 交互动作必须提供即时反馈（视觉或 Toast）。

## 7. 设计走查清单（提测前）

1. 视觉一致性：色彩、字号、圆角是否命中规范。
2. 交互反馈：可点击区域是否均有按压态/反馈。
3. 布局安全：是否遮挡 Tabbar、底部栏、刘海/圆角区域。
4. 异常态：空态、弱网、禁用态是否完整。
5. 文案质量：按钮文案是否为明确动作词。
6. 性能：是否避免重阴影、大面积高频动画。

## 8. 版本管理

- 规范文件路径：`/DESIGN_SPEC.md`
- 变更方式：PR 中必须附带“规范变更说明”与影响范围。
- 若设计稿与规范冲突，以“最新合并后的本规范”为开发基准。
